AI & content-usage policy

This page states, in plain language, how AI systems may use ledger.dosanjhlabs.com. Answer and search AI is welcome to crawl and cite this site — that drives real traffic to a free, open-source plugin. Automated collection of this site's content for AI/ML training, fine-tuning, or dataset creation is not permitted without a license.

Scope. This policy governs the content of this marketing/documentation website only. It has no bearing on the Ledger Governance plugin itself, which is separate open-source software released under the MIT License — read, fork, and use the code freely under those terms.

Answer & search AI — welcome

AI answer engines, assistants, and search crawlers (for example ChatGPT, Claude, Perplexity, Gemini/Google, and similar) may crawl, index, and cite this site in response to user queries, and may quote or summarize it with attribution and a link back to ledger.dosanjhlabs.com. robots.txt explicitly allows the current generation of answer/search bots: OAI-SearchBot, ChatGPT-User, Claude-SearchBot, Claude-User, PerplexityBot, Perplexity-User, Applebot, Amzn-SearchBot, Amzn-User, meta-webindexer, meta-externalfetcher, MistralAI-Index, MistralAI-User, and DuckAssistBot.

AI/ML training & dataset use — reserved

Automated collection of this site for the purpose of training or fine-tuning a machine-learning or AI model, building a retrieval corpus (RAG), or otherwise assembling a dataset from this content is not permitted without a prior written license from Dosanjh Labs. This is expressed in four machine-readable ways so both crawlers and rights-tooling can honor it:

  • robots.txt — known AI training/dataset crawlers (GPTBot, ClaudeBot, anthropic-ai, Claude-Web, Google-Extended, Applebot-Extended, Meta-ExternalAgent, FacebookBot, Amazonbot, CCBot, Bytespider, cohere-ai, cohere-training-data-crawler, Diffbot, Omgilibot, Omgili, webzio-extended, PetalBot, PanguBot, YouBot, ImagesiftBot, VelenPublicWebCrawler, Timpibot, Scrapy, and others) are disallowed.
  • X-Robots-Tag: noai, noimageai HTTP response header on every page.
  • Content-Usage: train-ai=n, search=y HTTP response header — machine-readable: search indexing is welcome, AI training is not.
  • W3C TDM Reservation Protocol — a TDM-Reservation: 1 / TDM-Policy response header pair and a /.well-known/tdmrep.json file reserving text-and-data-mining rights across the whole site.

Note: Google-Extended and Applebot-Extended are training-only opt-out tokens — disallowing them does not affect Googlebot or Applebot search indexing, which remain fully allowed.

Search indexing stays on

Nothing on this page or in the underlying robots.txt/_headers configuration blocks ordinary search indexing. This site wants to be found — it is answer-AI- and search-friendly by design, and only reserves the separate right of using its content as AI training data.

Licensing & contact

If you would like to license this site's content for AI/ML training, fine-tuning, or dataset use, get in touch via the contact form at dosanjhlabs.com/contact (please use the form rather than email).
