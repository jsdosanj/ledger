How it works in Obsidian

Governance that lives inside your vault.

Ledger Governance runs entirely within Obsidian. Everything it does is a Command Palette action, a ribbon dashboard, and a few standard frontmatter fields. There's no account and nothing that leaves your machine — it's 100% local, free, and open source.

It stays Obsidian, by design

Standard frontmatter

Governance is plain YAML — owner, status, reviewed_on, reviewer, review_cadence_days — fields Obsidian already reads natively. No proprietary store.

Portable Markdown

Every note stays a plain .md file. Disable or remove the plugin and your notes — and their governance fields — are untouched. No lock-in.

Local plugin data

Settings live in the vault's own plugin data. There is no account, no token, and no network call anywhere in the plugin.

Install it

Two ways, both quick. There's no build step and no account to create.

From Community Plugins

In Obsidian, open Settings → Community plugins, browse for Ledger Governance, install it, and enable it. That's it — the dashboard's ribbon icon appears and the commands are ready.

Open the Obsidian listing →

Manual install

Copy the plugin folder into your vault at <your-vault>/.obsidian/plugins/ledger-governance/ (it must contain manifest.json and main.js), then enable Ledger Governance under Settings → Community plugins.

The governance commands

All six live in the Command Palette (Ctrl/Cmd-P, type "Ledger"). The dashboard also has a ribbon icon.

CommandWhat it does
New governed documentScaffolds a note with the governance frontmatter already in place — owner, status: draft, type, reviewed_on, reviewer, and a review_cadence_days.
Set document statusMoves the current note through draft → in-review → approved → published.
Set document ownerAssigns an owner (name or email) to the current note.
Mark current note reviewedStamps reviewed_on with today's date and resets the staleness clock.
Governance dashboardOpens the vault-wide dashboard (also the ribbon icon): status mix, review freshness, and ungoverned docs. Click a row to open it.
Open governance audit logOpens _ledger/audit.md, the in-vault record of every governance action.

The governance dashboard

See the whole vault at a glance

The dashboard counts your documents and breaks them down by review freshness — never reviewed, stale, or fresh — and by lifecycle — draft, in-review, approved, published, or unset. It surfaces every ungoverned doc (missing an owner or a status) so nothing slips through.

A live status-bar summary

Obsidian's status bar keeps a running N docs · X stale · Y ungoverned count that updates as you edit frontmatter — a constant, quiet read on where your vault stands, no dashboard needed.

Staleness is judged per note: a document not reviewed within its cadence — 90 days by default, or a per-note review_cadence_days — is flagged stale.

An audit log you own

# every governance action appends one row to _ledger/audit.md

| date       | action   | document            | detail            |
| ---        | ---      | ---                 | ---               |
| 2026-06-27 | created  | Runbooks/Restore.md | status draft      |
| 2026-06-27 | owner    | Runbooks/Restore.md | → ops@example.com |
| 2026-06-27 | status   | Runbooks/Restore.md | → approved        |
| 2026-06-27 | reviewed | Runbooks/Restore.md | reviewed_on 2026-06-27 |

It's plain Markdown in your vault — append-only, readable in any editor, and entirely yours. The plugin never sends it anywhere.
