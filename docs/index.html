Ledger Governance — quick reference

Ledger Governance is a free, MIT-licensed Obsidian plugin that adds document governance — owners, review dates, and lifecycle status — inside your vault. It runs 100% locally: no account, no network calls, no servers. Governance lives in standard YAML frontmatter, so your notes stay portable plain Markdown.

Quick reference: the six commands

Open the Command Palette (Ctrl/Cmd-P) and type Ledger.

CommandWhat it does
New governed documentScaffolds a note with governance frontmatter already in place — owner, status: draft, type, reviewed_on, reviewer, review_cadence_days.
Set document statusMove a note through its lifecycle: draft → in-review → approved → published.
Set document ownerAssign an owner by name or email.
Mark current note reviewedStamps reviewed_on with today's date.
Governance dashboardAlso the ribbon icon (book). Shows total docs, how many are ungoverned, review freshness (never / stale / fresh), and lifecycle mix. Click any row to open that note.
Open governance audit logOpens _ledger/audit.md in your vault.

Frontmatter fields

Governance is stored in standard YAML frontmatter at the top of each note. Nothing proprietary — your notes stay portable Markdown.

Fields it uses

titleDocument title.
ownerWho is responsible — name or email.
statusLifecycle: draft / in-review / approved / published.
typeDocument type (your own category).
reviewed_onDate of the last review.
reviewerWho reviewed it.
review_cadence_daysPer-note review interval; overrides the default stale threshold.

How freshness is judged

A note not reviewed within the stale threshold (default 90 days, or its own review_cadence_days) shows as stale. A note never reviewed shows as never.

Ungoverned means missing an owner (when "Require an owner" is on) or missing a lifecycle status (when "Require a status" is on).

The status bar shows a live summary: Ledger: N docs · X stale · Y ungoverned.

Settings

Settings → Community plugins → Ledger. All settings are local, stored in your vault's plugin data.

Review-stale threshold (days)

Default 90. A per-note review_cadence_days overrides it for that note.

Require an owner

Toggle. When on, ownerless notes are flagged as ungoverned.

Require a status

Toggle. When on, notes without a lifecycle status are flagged as ungoverned.

Where the audit log lives. Every governance action appends a row to _ledger/audit.md — a plain Markdown table (date | action | document | detail). You own it; it never leaves your vault.
