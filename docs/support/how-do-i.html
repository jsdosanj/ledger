How do I…?

Short, task-focused recipes for the most common things you'll do with Ledger Governance. Each one names the exact command or setting. Open the Command Palette with Ctrl/Cmd-P and type "Ledger" to reach any command.

Jump to: Create a governed doc · Assign an owner · Move through the lifecycle · Mark a doc reviewed · Find stale or ungoverned docs · Read the audit log · Change the stale threshold · Require an owner or status

…create a governed document?

Run New governed document from the Command Palette. It scaffolds a note with governance frontmatter already in place — owner, status: draft, type, reviewed_on, reviewer, and review_cadence_days — so the note is governed from the moment it exists. See the feature guide for the full field reference.

…set or assign an owner?

Open the note, then run Set document owner and enter a name or email. This writes the owner field into the note's frontmatter, which is also what clears it from the "ungoverned" count when Require an owner is on.

…move a document through the lifecycle?

Open the note and run Set document status, then pick the next stage: draft → in-review → approved → published. The dashboard's lifecycle mix updates to reflect the change.

…mark a document reviewed?

Open the note you reviewed and run Mark current note reviewed. It stamps reviewed_on with today's date, which resets the note's freshness so it no longer shows as stale or never-reviewed. (You must have the note open first.)

…find stale or ungoverned documents?

Open the Governance dashboard from the Command Palette or the ribbon book icon. It lists total docs, how many are ungoverned, review freshness (never / stale / fresh), and the lifecycle mix. Click any row to open that note. The status bar also shows a live summary: Ledger: N docs · X stale · Y ungoverned.

…read the audit log?

Run Open governance audit log to open _ledger/audit.md — a plain Markdown table of every governance action (date, action, document, detail). It lives in your vault and is created on your first action. See Security & privacy for how it's stored.

…change the stale threshold?

Go to Settings → Community plugins → Ledger and adjust Review-stale threshold (days) (default 90). To override it for a single note, add a review_cadence_days value to that note's frontmatter and it will use its own cadence instead. See how stale is computed.

…make owner or status required?

In Settings → Community plugins → Ledger, turn on Require an owner and/or Require a status. With these on, any note missing an owner or a lifecycle status is flagged as ungoverned in the dashboard and the status-bar count.

Don't see a command? Make sure the plugin is enabled — see Troubleshooting. Many commands act on the active note, so open the target note before running them.
