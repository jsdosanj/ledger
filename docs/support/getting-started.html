Getting started

Ledger Governance adds document governance to Obsidian — owners, review dates, and lifecycle status — without taking you out of your vault. It is free, MIT-licensed, and 100% local: no account, no network, no servers.

What is Ledger Governance?

A free, local-only Obsidian community plugin for document governance.

Governance in frontmatter

Owners, lifecycle status, and review dates are written into standard YAML frontmatter at the top of each note.

Portable Markdown

Nothing proprietary. Your notes stay plain .md files you can read, edit, and take anywhere.

Free and MIT-licensed

Open source under the MIT License. No account, no token, no network calls.

Install

From Community Plugins

The simplest path. Use Add to Obsidian, then in Obsidian go to Settings → Community plugins, install Ledger Governance, and enable it. No build step, no setup.

Manual install

Copy the plugin folder to:

<vault>/.obsidian/plugins/ledger-governance/

The folder must contain manifest.json and main.js. Then open Settings → Community plugins and enable Ledger Governance.

Either way there is nothing to configure to begin — no account, no sign-up, no setup. The defaults work immediately.

Your first governed doc

Open the Command Palette (Ctrl/Cmd-P) and type Ledger to reach every command.

  1. New governed document — scaffolds a note with governance frontmatter already in place: owner, status: draft, type, reviewed_on, reviewer, and review_cadence_days.
  2. Set document owner — assign an owner by name or email.
  3. Set document status — move it along its lifecycle: draft → in-review → approved → published.
  4. Mark current note reviewed — stamps reviewed_on with today's date.
  5. Governance dashboard — also the ribbon icon (book). It shows total docs, how many are ungoverned, review freshness (never / stale / fresh), and lifecycle mix. Click any row to open that note.
  6. Open governance audit log — opens _ledger/audit.md, where every governance action is appended as a row (date | action | document | detail). You own it; it never leaves your vault.

The status bar keeps a live summary: Ledger: N docs · X stale · Y ungoverned.

How stale and ungoverned work

Stale & never

A note not reviewed within the stale threshold — default 90 days, or its own review_cadence_days — shows as stale. A note never reviewed shows as never. Use Mark current note reviewed to refresh it.

Ungoverned

A note is ungoverned when it is missing an owner (with "Require an owner" on) or missing a lifecycle status (with "Require a status" on). Both toggles live in Settings → Community plugins → Ledger, alongside the stale threshold.
