The Obsidian plugin — Ledger Governance

Ledger Governance is a free, MIT-licensed Obsidian plugin that adds lightweight document governance — ownership, a review-status lifecycle, stale detection, a dashboard, and an in-vault audit log — to notes you already keep. It runs entirely on your machine: no account, no token, no network.

100% local. Governance lives in your notes' standard YAML frontmatter, so they stay portable plain Markdown. Nothing is uploaded and nothing phones home. See Privacy.

What you get

The plugin adds six commands to Obsidian's Command Palette (Ctrl/Cmd+P, type "Ledger"), a ribbon icon, and a live status-bar summary. In short:

  • New governed document — scaffold a note with governance frontmatter already in place.
  • Set document status — move a note through draft → in-review → approved → published.
  • Set document owner — assign who is responsible for the note.
  • Mark current note reviewed — stamp today's date into reviewed_on.
  • Governance dashboard — totals, ungoverned notes, review freshness, and lifecycle mix at a glance (also the ribbon icon).
  • Open governance audit log — open the in-vault _ledger/audit.md record.

Each is detailed in The commands. For a full tour of the underlying fields and lifecycle, see Features.

Install

There is no build step and no account — Obsidian loads the plugin directly, and there is nothing to register. Pick whichever route suits you.

From Community Plugins (recommended)

  1. Use the Add to Obsidian link to open the listing in Obsidian's Community Plugins directory.
  2. Click Install, then Enable.

Manual install

  1. Copy the plugin folder into your vault at <vault>/.obsidian/plugins/ledger-governance/. The folder must contain manifest.json and main.js.
  2. In Obsidian, open Settings → Community plugins (turn off Restricted mode if needed).
  3. Find Ledger Governance in the installed list and enable it.

That's all. There is no account to create, no token to paste, and no setup screen. See Getting started for your first governed note.

The commands

All six commands live in the Command Palette (Ctrl/Cmd+P, type "Ledger"). The dashboard is also bound to the ribbon icon (the book in the left ribbon).

CommandWhat it does
New governed documentScaffolds a new note with governance frontmatter already filled in: owner, status: draft, type, reviewed_on, reviewer, and review_cadence_days.
Set document statusSets the active note's lifecycle status: pick draft, in-review, approved, or published.
Set document ownerAssigns the active note's owner — a name or an email address.
Mark current note reviewedStamps reviewed_on in the active note with today's date, resetting its review clock.
Governance dashboardOpens the dashboard (also the ribbon book icon): totals, ungoverned count, review freshness, and lifecycle mix across the vault. Click any row to open that note. See The dashboard.
Open governance audit logOpens _ledger/audit.md, the plain-Markdown record of every governance action. See The audit log.

The dashboard & status bar

The Governance dashboard (Command Palette or the ribbon book icon) reads governance frontmatter across your vault and summarizes it:

  • Totals — how many notes the plugin sees.
  • Ungoverned — notes missing an owner (when Require an owner is on) or a lifecycle status (when Require a status is on).
  • Review freshness — each note is grouped as never reviewed, stale (past its review threshold), or fresh.
  • Lifecycle mix — counts by status: draft, in-review, approved, published, and unset.

Click any row to jump straight to that note. The status bar shows a live one-line summary as you work — for example Ledger: N docs · X stale · Y ungoverned — so the health of your vault is always visible without opening the dashboard.

The audit log

Every governance action — creating a governed document, setting a status or owner, marking a note reviewed — appends a row to _ledger/audit.md inside your vault. It is an ordinary Markdown table:

dateactiondocumentdetail
2026-06-27reviewedRunbooks/Restore Internet.mdreviewed_on set to 2026-06-27
2026-06-27statusPolicies/Access Policy.mddraft → in-review

You own the file — it lives in your vault and never leaves it. Because it's plain Markdown, you can read, search, version, or archive it like any other note.

Settings reference

Open Settings → Community plugins → Ledger Governance. Every setting is stored locally in your vault's plugin data.

SettingMeaning
Review-stale threshold (days)A note not reviewed within this many days is flagged as stale. Default 90. A note can override it with its own review_cadence_days.
Require an ownerWhen on, notes without an owner are counted as ungoverned in the dashboard and status bar.
Require a statusWhen on, notes without a lifecycle status are counted as ungoverned.

Privacy

Ledger Governance is 100% local:

  • No network calls. The plugin never contacts a server. There is no account and no token of any kind.
  • Your data stays in your notes. Governance metadata is written into your own notes' frontmatter, and the audit log lives in your vault at _ledger/audit.md.
  • Settings stay local. They're saved in your vault's plugin data, on your machine.
  • Free and open source under the MIT License.

For more, see Security & privacy.
