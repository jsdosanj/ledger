Troubleshooting & FAQ

Plain answers for the common snags with the Ledger Governance plugin. Everything runs locally inside Obsidian, so most issues come down to the plugin not being enabled yet, or a command needing a note open first.

On this page: Plugin not showing up · Commands not found · "Mark reviewed" did nothing · Dashboard is empty · Where is the audit log? · How is stale computed? · Is anything sent anywhere? · How to uninstall

The plugin isn't showing up

If you don't see Ledger Governance after installing, work through these in order:

  1. Confirm the files are in place. The plugin folder must be at <vault>/.obsidian/plugins/ledger-governance/ and contain both manifest.json and main.js. If either file is missing, Obsidian won't list it.
  2. Turn off Restricted mode. Open Settings → Community plugins. If community plugins are restricted, click Turn on community plugins.
  3. Enable it. Under Settings → Community plugins → Installed plugins, find Ledger Governance and toggle it on.
  4. Reload if needed. If it still doesn't appear, close and reopen the vault (or run Reload app without saving from the Command Palette).

For the full walkthrough, see The Obsidian plugin and Getting started.

The Ledger commands aren't there

Open the Command Palette with Ctrl/Cmd-P and type "Ledger". You should see all six commands: New governed document, Set document status, Set document owner, Mark current note reviewed, Governance dashboard, and Open governance audit log. If nothing matches, the plugin probably isn't enabled yet — see Plugin not showing up above.

"Mark current note reviewed" did nothing

This command stamps reviewed_on on the active note, so you need a note open and focused first. Open the document you want to review, then run Mark current note reviewed from the Command Palette. If no note is open, there is nothing for the command to stamp. The same applies to Set document status and Set document owner — open the target note first.

The Governance dashboard is empty

Open the dashboard from the ribbon (the book icon) or via the Governance dashboard command. If it shows nothing, the usual reasons are:

  • No Markdown files yet. A brand-new or empty vault has nothing to report on. Create or open some notes first.
  • Everything is ungoverned. Notes without governance frontmatter still count toward the totals — check the ungoverned figure and the status bar summary (Ledger: N docs · X stale · Y ungoverned). Use New governed document, or add an owner/status to existing notes, to start populating the lifecycle mix.

See Every feature, step by step for what each dashboard row means.

Where is the audit log?

Ledger keeps an in-vault audit log at _ledger/audit.md. It is a plain Markdown table (date, action, document, detail) and is created on your first governance action — so if you haven't run any commands yet, the file won't exist. Run the Open governance audit log command to jump straight to it, or open _ledger/audit.md from the file explorer.

How is "stale" computed?

A note is stale when it hasn't been reviewed within the stale threshold. The default threshold is 90 days, set in Settings → Community plugins → Ledger → Review-stale threshold (days). A note can override this per-document with a review_cadence_days frontmatter value — that note then uses its own cadence instead of the global default. A note that has never been reviewed (no reviewed_on) shows as never rather than stale.

Is anything sent anywhere?

No. Ledger Governance is 100% local. It makes no network calls, has no account or token, and collects no telemetry. Governance metadata is written into your own notes' frontmatter, the audit log lives in your vault at _ledger/audit.md, and your settings are stored in the vault's local plugin data. Nothing leaves your machine. The full source is on GitHub if you want to verify it — see Security & privacy for details.

How do I uninstall it?

  1. Open Settings → Community plugins → Installed plugins.
  2. Toggle Ledger Governance off, then click the uninstall (trash) control to remove it.
  3. If you installed manually, you can instead delete <vault>/.obsidian/plugins/ledger-governance/.

Your notes are unaffected: the governance frontmatter and _ledger/audit.md are ordinary Markdown and stay in your vault. Delete them yourself if you no longer want them.

Still stuck? Open an issue on GitHub with your Obsidian version and what you tried — it's the fastest way to get help, and others benefit from the answer.
