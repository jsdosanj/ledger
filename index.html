Document governance for Obsidian

Govern your vault without leaving Obsidian.

Ledger Governance is a free, open-source Obsidian plugin that adds the layer a growing vault needs: a clear status lifecycle, an owner on every document, review tracking that flags what's gone stale, a governance dashboard, and an in-vault audit log. It's 100% local — no account, no token, no network. Governance lives in standard frontmatter (owner, status, reviewed_on), so your notes stay portable plain Markdown.

Obsidian is excellent for one person. The moment a team depends on those notes, you need to know who owns each one, whether it's current, and where it sits in its lifecycle. Ledger Governance adds exactly that — and nothing that takes your notes out of your vault.

What it does

Governed-document template

New governed document scaffolds a note with the governance frontmatter already in place — owner, status, reviewed_on, and a review cadence — so a document starts governed instead of being patched later.

Status lifecycle

Set document status moves a doc through draft → in-review → approved → published. Anyone reading the note can see exactly where it stands.

Ownership & review tracking

Assign an owner to any note, and Mark current note reviewed stamps reviewed_on with today's date. Docs past their review cadence (default 90 days) are flagged stale.

Governance dashboard

The ribbon icon opens a dashboard of your whole vault: the status mix, review freshness (never / stale / fresh), and which docs are ungoverned — missing an owner or a status. Click any row to open it.

In-vault audit log

Every governance action is appended to _ledger/audit.md — a plain-Markdown trail of what changed and when, that lives in your vault and that you own.

100% local

No network calls, no account, no token. Settings live in the vault's local plugin data. A breach of some server can't touch your notes, because nothing leaves your machine.

How it works in Obsidian

  1. Install it from Obsidian's Community Plugins directory and enable it. No account, no sign-up.
  2. Start a governed doc — run New governed document from the Command Palette (Ctrl/Cmd-P, type "Ledger"), or add the same frontmatter to an existing note.
  3. Assign an owner and set a status — the document now says who's responsible and where it is in its lifecycle.
  4. Mark it reviewed when you've checked it; the plugin stamps reviewed_on and tracks the cadence.
  5. Open the dashboard from the ribbon to see status mix, what's gone stale, and which docs are still ungoverned.
  6. Read the audit log any time — _ledger/audit.md records every action as plain Markdown.

It stays Obsidian, by design

Governance is just standard YAML frontmatter Obsidian already understands. There's no proprietary database and nothing to export.

  • Portable Markdown — every note remains a plain .md file you can open anywhere.
  • Standard fields — owner, status, reviewed_on, reviewer, review_cadence_days.
  • Live status bar — a running N docs · X stale · Y ungoverned summary.
  • No lock-in — uninstall and your notes (and their frontmatter) are untouched.

Questions, answered honestly

Is it really free?

Yes — free and open source under the MIT license. There's no paid tier, no trial, and no upsell.

Does it send my notes anywhere?

No. The plugin makes no network calls and stores no account or token. Governance metadata is written into your own notes, and the audit log lives in your vault.

Will I get locked in?

No. Governance is plain YAML frontmatter. Disable or remove the plugin and your notes — and their owner, status, and reviewed_on fields — are exactly as you left them.

How do I install it?

Open Obsidian → Settings → Community plugins, browse for Ledger Governance, install, and enable it. Or use the Add to Obsidian link. Manual install steps are in the docs.

Does it work on mobile?

Yes. It's not desktop-only — it runs wherever Obsidian does.

What counts as "stale"?

A document not reviewed within its cadence — 90 days by default, or a per-note review_cadence_days — shows as stale in the dashboard and status bar.
